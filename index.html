Projects

In my career, I have completed multiple projects that require strong logical and coding skills. Below is a list of a few projects that I have worked on, along with a brief description of each.

Forex Backtester

This project was done as a group project for a company by the name of Spatial Edge, as part of the Software Engineering module, which I achieved a distinction for. It involved building a web based application that could retrieve forex data, then use it to display a chart, along with a feature where the user can code a trading trategy on the site and run it across this data. The program will then display the chart data along with the performance of the strategy used, displaying entries and exits on the chart itself as well as a detailed summary of how the strategy performed. Other features includes version control of strategies, comparing versions showing insersions and deletions, as well as anomoly detection. This project consisted of a server-side and a client-side, utilizing React with Node.js, JavaScript and a bit of Python. CSS and Tailwind CSS was used for styling the front-end.

Co-Scribe

This project was done as a group project for Stellenbosch University in the Web Development module, which I achieved a distinction for. It involved building a web based application that allows users to create an account and log in, allowing them to choose a username and even a profile image, provided it was not already in the database, being used by another active user. Once users sign up and log in, they get access to a platform that allows them to create Notes. Notes, in this project, refers to a text file that can be worked on and edited in the text editor available on the platform. It converts whatever text is written into Markdown text, which is shown in a separate block beside the text editor. What makes this application special is that it allows users to share Notes with other users and allows users to collaboratively edit and work on these Notes together, at the same time, simultaneously, hence the name Co-Scribe. The platform also allows users to categorize these Notes into what can be thought as folders, in order to group certain Notes together, for example categorizing all Notes regarding Finance. It also allows users to search for Notes, as well as delete Notes, but keeps them in a trash just incase the user would like to revive it. Once it is deleted from the trash, it gets wiped from the database. This project made use of a server-side and a client-side, utilizing React with Node.js, Typescript, JavaScript and HTML. CSS and Tailwind CSS was used for styling the front-end.

Moving Maze

This project was done as an individual project for Stellenbosch University in the Object Oriented Programming module, which I achieved a distinction for. It involved building a Java application in the form of a game, which could be played through text on the terminal or through a GUI, depending on the arguments passed through to it when run. More information about the arguments are in the README.md file in the Moving-Maze repository. Instructions on how to play is given in the GUI mode. This project involved deep thinking to solve complex problems throughout the creation of the game, whether that be back-end or front-end. There is even a pathfinder AI that I have built into the game. You can ask this AI by choosing a position, if there is an open path to that specific position and it will either respond by placing you there, or by telling you that there isn't. This project, along with a few others that I have not mentioned, gave a nice full-stack experience, elevating my skills tremendously. The language used was only Java and utilized many .java files to practice OOP and its principles.

Stack Overflow Scraper

This project was done as an individual project for Stellenbosch University in the Web Development module, which I achieved a distinction for. It involved building a program that was able to scrape Stack Overflow along with Stack Exchange and search for information. The information it was searching for was provided by the university and they were things such as specific questions, specific answers, all questions in a filtered page, all answers on a specific question, etc. Within each of these categories, we had to gather a lot of different information such as the user, the date posted, whether an answer was accepted, whether the question is closed or locked, and many more (there are too many to list all). This was all done in Python using the BeautifulSoup API. The information was then displayed as a web page in the browser in the form of a JSON file, displaying information easily. The program uses various endpoints to determine what sort of information to look for, so the user should first understand which endpoints to use before using the program (this is shown in the code or in the README.md). This all takes place using Flask. So this program is a powerful tool for scraping an finding specific information on a website, using Python and Flask, along with the BeautifulSoup API.
